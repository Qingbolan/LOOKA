给你一版“完整的、基于文生图的时尚App”产品描述，形态上像小红书（内容流 + 收藏/种草 + 社交互动），但核心更偏“潮流时尚 + 生成式设计 + 可购买/可定制”的闭环。

产品名先用 Working Title：STYLO（你后面再改）。定位一句话：像小红书一样刷穿搭灵感，但每一条灵感都能一键变成“你的款”，并且能试穿、能下单、能拼单。

一、目标用户与使用场景
核心用户（第一阶段就能起量的）
1）潮流爱好者/学生党：喜欢街头、Y2K、Cleanfit、Gorpcore、机能风，追热点，愿意分享，预算敏感但愿意为“独特”买单。
2）内容创作者/KOL：需要高频产出穿搭内容；愿意用生成能力做“联名/胶囊系列”。
3）买不到合适尺码的人群：小个子/大码/特定体型，想“风格不妥协但要合身”。

高频场景

* 刷到一套穿搭想要同款但不想撞衫：生成“类似但更适合我”的版本。
* 明天要去面试/约会/音乐节：用自然语言快速出三套风格方案并试穿。
* 看到一张图（街拍/店铺图/明星图）：拍照找灵感 → 生成“同风格可买款”。

二、App 信息架构（像小红书，但每篇内容都是可操作资产）
底部 5 tab（建议）
1）首页 Feed：潮流内容流（穿搭/单品/主题/挑战赛）
2）生成 Studio：文生图/图生图的“设计工坊”
3）试穿 Closet：虚拟衣柜 + 试穿 + 搭配
4）商城 Shop：现货/定制/拼单/联名
5）我的 Profile：作品集、收藏、订单、收益、身份（创作者/设计师）

三、核心内容单位：LookCard（内容即交易的最小颗粒）
每条内容不是“笔记”，而是 LookCard，包含：

* 主图/短视频：穿搭效果（AI生成或真人穿搭）
* 风格标签：#gorpcore #cleanfit #y2k #minimaltech 等
* 设计配方 Recipe（可折叠显示）：prompt + 参数（版型/颜色/面料档位/细节）+ 参考图
* 试穿按钮：Try on me
* 购买按钮：Buy (现货) / Customize (定制) / Join Group (拼单)
* 二创按钮：Remix（基于同一 Recipe 生成你的版本）
* 互动：收藏、评论、转发、投票（更想要 A 还是 B）

这个设计让内容、生成、交易天然在一个卡片里闭环。

四、首页 Feed（仿小红书的刷法，但更“潮流/时尚”）
1）推荐逻辑（用户能感知到“懂我”）

* 基于：收藏/停留/Remix/下单/试穿记录
* 维度：风格（style）、场景（occasion）、版型偏好（fit）、色系（palette）、价位（budget）、城市天气（可选）

2）Feed 里的三类内容
A. Trend Drop（潮流热榜）：每周“风格胶囊”，例如“机能风一周穿搭”
B. Creator Look（创作者作品）：更强的个人签名风格
C. Brand/Shop Look（品牌店铺）：品牌可发布可买 LookCard，但必须可 Remix（更像“可互动广告”）

3）潮流玩法（增强时尚属性）

* Trend Challenges：主题挑战赛（“全黑机能”“复古球衣”“Cityboy通勤”）
* Drop Culture：限时“胶囊系列”发售 + 拼单倒计时
* Fit Battle：同一主题 A/B 两个 look 让用户投票，胜者上榜并获得流量

五、生成 Studio（文生图是核心引擎，但要“时尚化 UI”）
1）输入区（像和造型师对话）

* 文本：我想要一套“黑色机能风”，适合新加坡通勤，下雨也OK，预算 80-120，新手别太夸张
* 结构化补充（可选）：风格/场景/预算/面料偏好/不想要的元素（露腰、太紧、太花等）
* 参考图：上传街拍图/明星图/店铺图（作为风格参照，不强调复刻）

2）输出区（不是单张图，是“候选系列”）

* 一次出 6 张：同风格不同细节（领型、版型、裤型、配饰）
* 每张都能点开编辑：颜色、长度、廓形、元素（口袋、拉链、反光条）
* 生成质量开关：快速/高清（会员点数）

3）Remix 工作流（关键）

* “保留风格、改配色”
* “保留上衣、换裤子”
* “更日常/更夸张”
* “更显瘦/更显高”
  这些是产品化按钮，降低普通用户提示词门槛。

六、虚拟衣柜 + 试穿 Closet（留存模块）
1）衣柜录入

* 拍照识别：T恤/衬衫/外套/裤子/鞋/包
* 电商订单导入（后续做）
* AI 自动补全属性：颜色、版型、材质、风格

2）试穿

* MVP：2D试穿（把生成服装贴到人像）+ 体型优化（显高/显瘦建议）
* 进阶：3D avatar + 尺码建议 + 面料垂坠感

3）搭配与提醒

* “一件多穿”：你这件外套可以搭你衣柜里 6 条裤子
* “缺口提醒”：你缺一双能搭 70% look 的鞋
* “天气穿搭”：结合新加坡天气（可选）

七、商城 Shop（把潮流属性做成“Drop + 拼单 + 定制”）
商品分三层（非常重要）
1）现货层：快速成交（平台精选/品牌合作）
2）轻定制层：改色/改长度/改印花（低风险）
3）深定制层：小单快反（拼单达 MOQ 才生产）

购买路径（单次不超过 3 步）

* Try on → 选择尺码/改动 → 下单（现货）
* Customize → 选择改动幅度 → 预售定金（定制/拼单）

潮流运营机制

* Weekly Drop：每周 10 个精选 look 上架，限时拼单
* Creator Capsule：创作者胶囊系列，粉丝拼单
* 城市风格馆：Singapore street / Seoul vibe / Tokyo tech 等

八、创作者体系（像小红书，但创作者能赚“设计钱”）
创作者分级

* 普通用户：发 LookCard、参与挑战赛
* 认证设计师/潮流创作者：可发布 Capsule，可设定分成
* 品牌店：可投放但必须“可 Remix”，形成互动传播

收益模型（对创作者更有吸引力）

* 数字配方售卖（Recipe Pack）
* 实物成交佣金
* 拼单成功奖励
* 品牌合作任务（平台撮合）

九、增长与传播（决定你能不能像小红书一样滚起来）
天然传播点

* “Remix 生成我的版本”是比“收藏笔记”更强的社交行为
* 每个 LookCard 有可分享的动态卡片：图 + 风格标签 + 一句话配方
* 挑战赛榜单：上榜即流量

冷启动策略（务实）

* 先做 3 个风格大类（机能/cleanfit/Y2K），每类 200 条高质量种子内容
* 合作 20-50 个小体量潮流博主，做“联名胶囊 + 拼单”
* 把“明天穿什么”做成每日任务（提高日活）

十、定价与商业模式（你需要明确赚钱点）
基础版：免费（每天有限生成次数 + 基础试穿）
会员：更多生成次数 + 高清 + 高级试穿 + 衣柜高级功能
交易抽佣：现货/定制每单抽佣
品牌广告：不是传统 banner，而是“可 Remix 的 Look 投放”
创作者分成：数字配方与实物佣金

十一、MVP 范围（不做什么比做什么更重要）
MVP 只做：

* Feed（可刷、可搜、可收藏）
* Studio（文生图 + 少量可控编辑按钮 + Remix）
* Try-on（2D 试穿）
* Shop（先做轻定制或预售拼单其一）
* 分享（卡片分享 + 评论/收藏）

先不做：

* 全3D物理模拟
* 全品类（先专注一个品类：上衣或连衣裙或外套）
* 复杂社交（私信/群聊先别碰）

十二、你要的“较强潮流时尚属性”如何在产品上被感知
1）视觉语言：整体像“潮流杂志 + Drop 文化”，不是工具软件
2）内容组织：热榜、挑战赛、胶囊系列、风格馆，强化潮流叙事
3）交易机制：限时 Drop + 拼单，让“抢到/参与感”成立
4）创作者生态：联名胶囊系列比普通种草更潮
